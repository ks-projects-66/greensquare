Methodology

How the benchmark was run.

The headline is the reversal. This page explains the test behind it, including what it does and does not prove.

Back to the evidence

Design

Matched inputs, two conditions.

The test compared a baseline response against a GreenSquare framework response. Each run used the same fictional business case. The baseline answered the case directly. The framework run followed the GreenSquare decision structure before producing the brief.

PlatformsClaude, ChatGPT and Gemini
InputsOne fictional case, used consistently
ScoringOutputs scored against the same rubric
PurposeTo test whether better structure produces a more decision-ready answer

What changed

The input did not change. The structure did. The framework forced the decision to be tested before the recommendation was written.

Rubric

Twelve dimensions.

The five categories on the evidence page are a plain-language summary of these dimensions.

#DimensionWhat it measures
1Structural completenessCoverage of the sections needed to make the decision usable

11Extraction qualityAbility to surface information the user may not volunteer
12Adaptive intelligenceAdaptation to the specific context

Result

A consistent lift, with the reversal doing the real work.

The framework moved output from competent to decision-ready. A better answer is useful. Catching that the question is wrong is the point.

AI baseline average~6.1 / 10
GreenSquare framework average~8.2 / 10
Framework lift~35 to 45%

Caveats

Useful evidence, not statistical proof.

  • The company is fictional.
  • This is a controlled benchmark, not a live field trial.
  • The sample is small.
  • Scoring involves judgment, even with a consistent rubric.
